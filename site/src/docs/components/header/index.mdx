### Principles

- **It is strongly recommended to always include the header component on every page in your service.**
- The Header component is a wrapper for the HDS header navigation components. This means you should put all the other header elements inside this component.
- The only mandatory component inside the header is HeaderActionBar. Other components may be used if needed.
- The inner components should **always** be in this order: Header.SkipLink, Header.UniversalBar, HeaderActionBar and HeaderNavigationBar.
- Try to keep the order of actions (language select, search, log in) consistent in navigation. Do not change the default order without a good reason.

### Variations

#### With action bar

- **You should always include the Header.ActionBar component in your Header**.
- The Header.ActionBar component provides the service logo, name and possibly language selection, search and log in features.
- On smaller screens (768px and down) the Header.ActionBar undergoes some visual changes:
  - The contents of Header.NavigationMenu and Header.UniversalBar, if you have provided either of them, will move inside Header.ActionBar's menu.
  - The Header.NavigationLanguageSelector moves under the ActionBar.

#### With navigation menu
- The Header.NavigationMenu component should come after [the Header.ActionBar component](#with-action-bar).
- Keep navigation menu link labels clear and concise. Prefer max 1-2 word long labels. Avoid starting multiple menu link labels with the same word. See also our <InternalLink href="/components/header/accessibility">accessibility notes</InternalLink> about navigation menu's hierarchy.
- The component supports up to 2 levels of nested links. 
- On smaller screens (768px and down) the Header.NavigationMenu is hidden and the contents are moved inside the [the Header.ActionBar component's](#with-action-bar) menu.

#### With universal bar

- Using the Header.UniversalBar component is optional. Use it to provide frequently visited and useful links.
- If the primary link information is provided, it should direct to the webpage of City of Helsinki.
- Keep the link labels clear and concise. Prefer max 1-2 word long labels so the labels don't go to the next line on smaller screens. Avoid starting multiple link labels with the same word.
- On smaller screens (768px and down) the Header.UniversalBar is hidden and the contents are moved inside the [the Header.ActionBar component's](#with-action-bar) menu.

#### With navigation language selector

- Using the Header.NavigationLanguageSelector is optional. Use the navigation language selector to define site languages. 
- The Header component has `onDidChangeLanguage` prop function, where user can define a callback function which gets the selected language key (value) as parameter.
- See example for Header.NavigationLanguageSelector <Link openInNewTab openInNewTabAriaLabel="Opens in a new tab" href="/storybook/react/?path=/story/components-header--minimal-with-localization">in Storybook</Link>
- On smaller screens (768px and down) the Header.NavigationLanguageSelector moves under the Header.ActionBar.

#### With skip link

- Using the Header.SkipLink component is optional. Use the skip link component to help keyboard-only users skip to the main content on a page.
- The Header.SkipLink supports `ariaLabel` prop which gives possibility to define more informative text for screen readers.

#### With search
- Using the Header.NavigationSearch is optional. It uses the <InternalLink href="/components/search-input">HDS SearchInput component</InternalLink>.
- The input value can be obtained via `onChange` callback prop as the input value changes, or `onSubmit` callback prop which is called when the search icon is pressed.

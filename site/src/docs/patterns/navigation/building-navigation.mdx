## Recommended order of navigation components

When implementing a navigation pattern, it's essential to consider the order of navigation components to ensure a logical, intuitive, and user-friendly experience.

### 1. Header
The header is placed at the very top of the page and serves as the primary navigation area for your website or application. It includes the City of Helsinki logo, site title, and main navigation menu. <strong>The Header component is a wrapper for the HDS header navigation components.</strong> This means you should put all the other header elements inside this component.

The header's consistent placement and appearance across all pages make it easily recognizable and accessible to users, allowing them to navigate the site quickly and efficiently.

        <Header.NavigationLink href="#" label="Link 3" />

  </li>

### 2. Breadcrumbs
Breadcrumbs are a secondary navigation component that helps users understand their current location within the site hierarchy and offers a simple way to navigate back to previous pages. Placed immediately below the header, breadcrumbs provide context and enhance usability by offering a clear path through the website or application.

<ul style="margin-bottom:0;">
  <li>
    The learn more about the <strong>Breadcrumbs</strong> component, please refer to<InternalLink href="/components/breadcrumbs">the component page.</InternalLink>
  </li>
</ul>

### 3. Hero
The hero section is an optional but highly recommended component that usually appears below the breadcrumbs. It serves as a visual and content-focused introduction to the page, typically including a large image, heading, and a call-to-action button or message. The hero section helps grab the user's attention and sets the tone for the rest of the page.

<ul style="margin-bottom:0;">
  <li>
    The learn more about the <strong>Hero</strong> component, please refer to<InternalLink href="/components/hero">the component page.</InternalLink>
  </li>
</ul>

### 4. Main Content
Following the hero section, the main content area is where the primary information and functionality of the page are presented. This section should be well-structured, with headings, subheadings, and other design elements that guide users through the content. The main content area may also include additional navigation components, such as side navigation or tabs, to facilitate access to related content or functionality.

### 5. Footer
The footer is placed at the bottom of the site and includes supplementary information and links, such as contact details, legal information, social media icons, and secondary navigation elements. The footer helps users find additional resources and reinforces the overall site structure by providing a consistent and accessible navigation area on every page.

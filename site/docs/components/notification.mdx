# Notifications

<LargeParagraph>
    Notifications are used to present timely information to the user. HDS offers two types of notifications for different use cases.
</LargeParagraph>

## Principles
- Notifications are designed to capture users' attention. **Therefore, they should be used sparingly only for important information or updates.**
- **Notification title should include the most important information about the notification.** Aim for title that is short, concise and easy to understand. Avoid titles that span over multiple lines.
- Use notification levels and colours consistenly. You can find guidelines on when to use each status colour in the [colour guidelines](/design-tokens/colour#ui-colours "UI colours").
- HDS offers multiple options for notification screen locations. Choose locations that fit best for your service and stay consistent.

### When to use each notification type?
- **Use inline notifications when information is related to the content or if you want to keep information visible at all times.** Inline notifications are part of the content and therefore they can be placed as close as possible to the related part of the content.
- **Use toast notifications to inform the user about a specific event or change in the system status.** Toast notifications do not relate to any specific object on the page and therefore they are placed on top of the content (usually top right or bottom center).

## Accessibility
- Avoid using automatically disappearing toast notifications for critical information. It is possible that the notification disappears before the user or screen reader can react to the message. Refer to <Link href="https://www.w3.org/WAI/WCAG21/Understanding/no-timing.html" external>WCAG 2.1 No timing requirement</Link> for more information.
- Remember that colour should never be the only way of conveying information. Make sure the meaning of the notification is clearly described by the title. Refer to <Link href="https://www.w3.org/WAI/WCAG21/Understanding/use-of-color.html" external>WCAG 2.1 Use of Color guideline</Link> for more information.

Colour

HDS colours

Helsinki Design System uses the colours from the City of Helsinki brands. It has a built-in default theme but also provides means to create new themes with little effort. When creating new themes, it's important to use the brand and UI colours defined in the system. For ease of use, these are provided as CSS and SASS variables in hds-design-tokens.

Colour accessibility

We are committed to complying with AA standard contrast ratios. To do this, choose primary, secondary, and extended colours that support usability by ensuring sufficient colour contrast between elements so that people with low vision can see and use the interface. If you create a new theme, you need to ensure that it is accessible.

Using colours

Color theming

The service design uses colour themes including three colours with grayscale and ui-helper colours. The colors are defined as CSS Variables and are therefore customisable. In addition to the base colours, the schemes provide lighter and darker shades for the colours.

  • Primary colour is the color displayed most frequently across your services and components. If you are not using a secondary color, your primary color can also be used to accent elements.

  • Secondary and tertiary colours provide more ways to accent and distinguish your service. The secondary and tertiary colours should be applied sparingly to accent select parts of your UI.

  • UI colours consist of the neutral colour shades as well the common colours for feedback (error, warning, success, info).
